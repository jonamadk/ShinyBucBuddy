Retrieval Quality Log

Top 5 retrieved documents (pre-rerank) vs Top 3 re-ranked (passed to LLM) — per prompt.

How retrieval is measured
Top 5 Retrieved = documents pulled from ChromaDB before reranking (chat_history.top_5_retrieved)
Top 3 Re-ranked = documents after cross-encoder reranking, passed to GPT-4o-mini (chat_history.top_n_document)
Citations Shown = what was actually displayed to the user (chat_history.citation_data)
Retrieval Accuracy KPI = % of queries where top_n_document is non-empty (i.e. at least 1 doc made it to the LLM)

User Feedback Score

Every thumbs up / down and written comment — tied to the exact query and response.

How feedback score is calculated
Feedback Score (%) = Thumbs Up ÷ (Thumbs Up + Thumbs Down) × 100
Table: chat_feedback · Fields: vote (up/down) · comment · userquery · llmresponse · timestamp
Every row below is one user rating. You can verify the total by counting the rows and comparing to the KPI on the Overview page.

Guest (Unauthenticated) Sessions

Queries from users who did not log in — tracked separately.

How guest vs auth is determined
Guest query = chat_history row where useremail IS NULL
Auth query = chat_history row where useremail IS NOT NULL
Auth % = COUNT(useremail IS NOT NULL) ÷ COUNT(*) × 100
Unique Guest Sessions = COUNT(DISTINCT guest_session_id) from guest rows
Returning Guest = same guest_session_id appears in 2+ separate visits
guest_session_id is generated in the browser (localStorage) and sent with every unauthenticated request — persists across page refreshes until cookies are cleared.
